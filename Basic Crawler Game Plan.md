### Basic Crawler Game Plan

Plan dungeon crawler game. 

##  Description

The should take influence from the famous Nethack game. A Rogue like dungeon crawler in a random generated dungeons with turn based fights, resource management, permanent death.


## Engine

The game will be implemented by the Godot 4.6.3 game engine

## Influence

The game will be highly influenced by OSR TTPRPGs. Especially, Basic Fantasy

## Graphic

The game graphic should resemble the ASCII based graphic of Nethack and Rogue. The main character will be visualized by the '@' character, monsters will be represented by the English ABC etc... But the game will not use ASCII. Instead, the charactres will be grahpic character that will imitate the ASCII character. The movement should be smooth and not like ASCII games where that character disappear and change to the next cell in the grid. 

## Layout

* The layout should resemble nethack, The name of the character, stats, AC, XP, Level should appear at the button of the screen. The dungeon on the middle of the screen. Messages are on the top of the screen

## Diffuclty

The game should not be very difficult. 

## Suppoerted platforms

The Godot project is configured to support Desktop, mobile and web games. Try not to break it.

## Notes

There is already a ready Godot project in the folder. Use it.

## Gameplay

* In the later stages of the game, the player could select race, class and roll for stats (Strength, Dexterity, Constitution, Intellgent, Wisdom, Charisma). At this early stage, It will be human figher 
* The character will be controller by the keyboard arrows by default
* Not like Nethack, the user will be able to see the dice rolls when attacking or saving. Not when looking for traps or secret doors. 
* We will start with just a few monsters and we will add more later on
    * S - Snake
    * R - Rat
    * s - Spider
    * G - Goblin

## Workflow
* Separate the work to smaller stages. For examle, start with the random dungeon generator and then move to the next phases after I will check them.
